# agent四项检测指标

1. tool call accuracy

   定义是工具调用准确度，一共有四个参数构成

   - sequence_aligned 严格模式下，全对 1 错一个 0 长度不同自然也是0  非严格模式 ， 调用的工具全就可以，但是不能多调用也不能少调用，顺序无所谓 所以在这里对于coverage_penalty没用的，调用的不一样就是0

   - arg_score  调用一个工具参数对了几个，描述调用一个工具正确的程度

   - coverage_penalty 

     - 如果调用的工具少了才启用 = 实际调用的工具数目/预期的工具数目   
     - 或者 = min(len(实际调用)，len(预期调用)) / len(预期调用)

   - final_score 
     $$
     final\_score =\frac{\sum arg\_score}{len(ref\_tools)} * coverage\_penalty * sequence\_aligned
     $$
     

2. tool call f1

   同样是有四个参数

   - TP = |actual $\cap$ expected|

   - FP = |actual - expected|

   - FN = |expected - actual|

     P = $\frac{TP}{TP + FP}$ R = $\frac{TP}{TP + FN}$ 

   - F~1~_score = $\frac{2PR}{P+R}$ = $\frac{TP}{\frac{expected + actual}{2}}$

     真正调用的工具和预期调用的工具的交集数目 / 预期和实际调用的均值





3. agent goal accuracy

   - 整理对话过程为可读字符串
   - 借助LLM将这段字符串分析出来一个end_state，这个也是一个字符串，用来描述执行的结果
   - 借助LLM将end_state和对应的参考的结果reference进行比较，得到一个verdict，这个只有0.0和1.0两种可能即完成和未完成
   - float(verdict)作为最终结果

   

   


4. topic adherence

   1. 围绕主题的程度
   2. 步骤
      - 对话转string，LLM抽取topics
      - 每个topic都用LLM，判断是否回答了，得到数组answered[bool]
      - 用LLM将topics与reference topics归类得到in_ref[bool]
      - TP = $\sum(answered \& in\_ref)$
      - FP = $\sum(answered\& \not in\_ref)$
      - FN = $\sum (\not answered \& in\_ref)$
      - P = $\frac{TP}{TP+FP+\epsilon}$
      - R = $\frac{TP}{TP + FN + \epsilon}$
      - $f1\_score=\frac{2PR}{P+R+\epsilon}$

# 标准选择原因

1. Tool Call F1
   1. **数据和指标的高度契合**：本数据集同时提供了 `steps/tool_call/input` 与 `expected_steps` 两类信息，能够自然构造“模型实际工具调用集合”与“参考工具调用集合”的对照关系。因此，Tool Call F1 在数据结构层面与任务标注高度契合，计算过程清晰且可复现。
   2. **刻画更加全面和稳健**：相较于仅使用单一准确率指标，F1 更能稳健刻画工具调用质量。工具使用场景中常见“遗漏调用”（FN）与“冗余调用”（FP）两类误差，F1 同时综合精确率与召回率，能够更全面地反映代理在工具规划与选择上的真实表现。
2. Agent Goal Accuracy
   1. **数据和指标的高度契合**：Agent Goal Accuracy 直接衡量“任务目标是否被达成”。数据集中包含 `ground_truth`（期望达成的目标/结果）以及完整执行轨迹（`steps + observation + final_answer`），适合将每个样本归约为“成功/失败”的二分类（0/1）判定，从结果层面评价代理的有效性。
   2. **该指标与 Tool Call F1 形成互补**：Tool Call F1 关注过程层面的“工具调用是否与参考一致”，但工具调用一致并不必然意味着任务成功，反之亦然。Agent Goal Accuracy 从最终结果层面评估成功率，与 Tool Call F1 共同覆盖“过程正确性”与“结果正确性”，从而提供更完整、立体的代理能力评估框架。
