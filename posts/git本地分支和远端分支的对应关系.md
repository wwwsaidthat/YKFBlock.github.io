# Git&GitHub&GitHub Desktop

1. 工具使用背景：

   1. 在程序员编写代码的时候往往不是一个人单独进行编程，往往是由一个团队多个程序员协同工作的，为了实现程序员之间的协同工作，产生了工具git 和github来进行管理代码版本和协同工作，但是由于用git对github上的代码仓库进行操作往往会遇到各种各样的问题，比如网络连接或者操作指令等问题，所以在这里笔者推荐一个工具--github desktop 真的很好用 狠狠安利！！！！

2. 阐明一下这三者的关系

   1. github是一个网站，注册之后用来存储程序员的代码，有点类似于百度网盘存储一个个文件夹，文件夹下面有文件，这个github存储的是一个个仓库，一个个仓库下面也是有各种各样的文件，就好比文件夹下面的各个文件
   2. git是用于本地仓库和远程的github上的仓库构建起来联系的一个终端工具
   3. github desktop是一个可视化git功能的一个软件，可以帮助你有效的理解git里面的每一条指令都是什么含义

3. 阐明一下一个真正的团队协作的过程是什么样的，都需要什么功能以及怎么用github来支持相应的功能

   1. 团队协作的过程

      1. 一个项目可能分成很多个部分，每个小组负责一个部分，一个小组中的每个成员分别负责这个部分的不同板块，有的是共同开发同一个功能，有的是开发不同的功能并发
      2. 这就需要涉及下面几种情形，这个软件就需要提供针对这几种情形相应的功能支持
         1. A在C代码基础之上增加了a B在C代码基础之上增加了b，管理者需要把ab都加到C里面去
         2. A改了C中的A成A' B改了C中的A成A'', 管理者需要选择一种方式把A'和A''留下来

   2. 功能 git工具的命令对应的github destop里面的实现方式

      1. 分支

         1. 分支定义 分支上的节点就是一个版本，每个分支往往会进行各个不同的功能的开发，最终合并成一个分支作为最后的结果

         2. 本地分支 存储在你本地的电脑上，只有你可见可修改

         3. 远端分支 存储在远端服务器，开发团队中所有人都可见，可以拉取

         4. 二者之间的联系记录：本地分支与远端跟踪分支的**跟踪关系**，是通过 `.git/config` 文件中的 `[branch "分支名"]` 配置段实现的，记录了 “本地分支 ↔ 远端别名 + 远端分支” 的映射；

         5. 跟踪关系：所有跟踪关系的建立、修改、删除，本质都是修改本地 `.git/config` 配置，和远程服务器无关；`git pull/push` 时，Git 会读取这个配置来自动确定操作的远端和分支。

         6. 提交分支

            1. 本地：工作区 暂存区 本地仓库
               1. 工作区你是用来修改你自己的代码，你自己怎么改都行，git不会跟踪这些改变
               2. git add . 把所有修改从工作区转移到暂存区，git开始跟踪这些修改
               3. git commit -m "message"把暂存区的内容转到本地仓库，并加上message信息说明
               4. 本地仓库里面会有本地的分支也会有远端的分支，本地仓库（`.git` 目录）通过两个目录存储分支指针：`refs/heads/` 存本地分支，`refs/remotes/` 存远端跟踪分支；

         7. 拉取分支

            1. git pull是拉取远端分支并与本地分支合并的过程 相当于git fetch & git merge
               1. git fetch 就是修改本地仓库里的远端的节点 把最新的提交更新到本地仓库里面，不改变工作区
               2. git merge 合并把本地仓库和远端拉取的最新分支合并

         8. 合并分支

            1. 可以在本地合并完了再提交 适合个人开发

            2. 也可以在github上面实现这个功能

               步骤 1：在 GitHub 上创建 PR

               1. 打开你的 GitHub 仓库页面，点击顶部的 **Pull requests** → **New pull request**；
               2. 选择 “源分支”（base）为 `main`，“目标分支”（compare）为你推送的 `feature/login`；
               3. 填写 PR 标题（如 “新增登录功能”）和描述（说明修改内容），点击 **Create pull request**。

               步骤 2：代码评审（可选但推荐）

               1. 邀请团队成员查看 PR，成员可以在网页端直接评论代码、提出修改建议；
               2. 如果你需要修改，直接在本地 `feature/login` 分支修改后再次推送：

               步骤 3：合并 PR 到远端分支

               1. 评审通过后，在 PR 页面点击 **Merge pull request**（合并按钮）；
               2. 选择合并方式（GitHub 提供 3 种常用方式，新手选默认的 **Create a merge commit** 即可）；
               3. 点击 **Confirm merge**，GitHub 会自动把 `feature/login` 合并到 `main` 分支；

4. github desktop上面的功能可以直接通过按钮点击的方式实现，你每次clone下来进行修改之后会自动检测变更，把变更提交到本地仓库，你点push origin自己就帮你交上去了很方便的！！！
