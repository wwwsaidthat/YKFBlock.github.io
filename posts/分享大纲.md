# 分享大纲

1. 操作系统
   1. 首先我想说说听课，说实话对于大学生来说集中特定的上课那两个小时来一直听课太难了，如果能做到当然最好，对于做不到的绝大多数的同学，我想谈谈这门课怎么听：
      1. 课上的时候，葛老师是一个很认真的人，我们那届有节课被假期还是什么占掉了，葛老师在期末的时候线上把这节课的课时给我们补回来了。另外就是葛老师讲课比较学术，知识点密度极大，尤其是对于一门需要一定逻辑思维的课程，听着听着容易跟不上，接下来就溜号，所以就要有选择的听：
         1. 每节课上课老师会提问，提问的内容是上节课的重点或者对于上节课的理解，这部分不管你会不会，老师提问完把问题和答案一字不拉的记下来，所以每节课上正式课之前的提问一定要认真听
         2. 剩下的就是难点听一听，没听会就算了，知道有这样一个内容就好了，课后我学会学懂就行了
      2. 课程的内容上，这学期的OS和上学期的计组会有部分知识重合，就比如那个磁盘计算，但是又有不同的侧重点，同时注意一下会有大块的难点，期末考试会占很大的分数，所以要想拿一个好分数，要弄透
      3. 另外知识点比较碎，每个大章内的内容又会有很多细小的知识点，解决这种学科最好的办法就是做思维导图，每章梳理一个，把细小的知识点都整理上去，我自己在学的时候梳理过，一共有五个PDF，上面是我自己的一些理解，因为里面有一些理解都是我自己想的我自己好理解的方式，写出来的东西很不官方，有很多描述可能会有一些大白话，或者东北话，甚至可能会理解上有一些偏差，所以我就不发到群里了，有想要的可以加我，我在你们QQ群里，读不明白的可以问问你们身边的东北同学哈哈。虽然我整理过一遍了，但是我的建议是你们自己学完每章内容梳理一下这章都有什么内容，这样在你自己的脑海里形成一张表，考试的时候你就照着你自己脑子抄就行了，我自己都是期末整理的，但是我的建议是更好的方式学完一章整理一下，期末很忙，忙不过来，当然你觉得突击效果好的话，就按自己节奏来
      4. 另外就是PV操作这里是个我上面说的大难点，期末考试会考一道大题分值很大10+分。你如果没学明白，真的写不出来，而且这个是临近期末的时候学，所以要下点功夫，找同学问，找老师问，因为我期末考试的时候看到有同学那个题一个字都没写，我当时也非常能理解他那种看天书的感觉，不会你是真写不出来。
   2. 重点！！！ 千万千万要好好做操作系统的作业，我当时以为跟别的课一样交了就是满分，但是这个课的助教会批，批分的那种，然后根据分给平时分，我自己做的时候，我就没太追求正确率，不会的问问同学，会的也没核对一下，等期末拿出来复习的时候才发现那时候理解错了，而且我平时分被扣了两分，就很不值当。
   3. 最后说说这个课的实验部分，可以找老师可以找助教，当然了老师问的问题可能会更深点，所以说你在不确保自己完全理解透了的情况，可以优先选择助教们，手册写的比较细，大家照着手册一步一步做就能完成，这部分尽量拿满分，比较容易的。
2. 软工2
   1. 这个系列课程其实并不是我的强项，真的很让我头疼，邢老师让我分享的时候我还觉得有点不知所措，后来邢老师说跟大家分享分享踩雷经验，我想这也是好的，关于这颗雷我谈谈自己的看法
      1. 首先67老师太和蔼了，所以相对比较看自觉，而且上课的内容偏向于工程理论，我自己觉得有点无聊，所以上课的时候并没有认真听，全靠期末的突击，背了两天书就上考场了
      2. 考试2个小时，18页卷子，题实在太多了，而且还不知道答什么，那时候我可能连对象和类都分不清（上学期学C++才学明白），去答那个题是在太难了，所以学好软工二的基础是软工一，软工一是你能看懂软工二题目的基础，今年据说又改革了，希望改简单点，至少能答完卷子
      3. 另外我思考我觉得这个课程无聊的本质原因是我的工程实践太少了，考完期末考试我和我的比较要好的朋友出来抱怨说觉得这个课程理论部分实用性太低了，他跟我说他觉得还挺有用的，事实证明他是对的，他后来那个暑假去字节实习，觉得这些理论是能用上的，所以这个课程知识对目前的学习没什么大的指导意义，未来真正坐到大厂里的时候你会认清这些知识的用处的
      4. 关于实践部分，我们那年做番茄商城，这应该是你们做一个完整的网页的一次体验，当然这个项目做好了，可以写到简历里面，虽然已经烂大街了的项目，如果你对其中使用的技术很熟悉，写到里面，面试官问的话也是可以的，毕竟现在还没什么经历可写的，因为我那个去字节实习的朋友就是这么拿到offer的
      5. 所以总结一下，这个课程不是一个短期见效涨能力的课程，这是一个长期投资，投入的精力在未来也会回报给你的
3. 小组合作
   1. 最后谈谈关于小组合作这个问题，我会从选队友，合作方式两个维度上谈谈
   2. 作为软工的学生，相信大家在大一的时候就已经被各个老师告知要学会合作，软工二课堂上也有相关管理合作方式的讲解，大家上课时可以仔细听听，而且未来的大三课程，几乎所有的课程的考核方式都是组队考核，未来进入工作岗位，也会需要你去跟同事交流合作，所以要有一个强大的小组合作能力。
   3. 选队友找熟人还是找生人？
      1. 熟人：
         1.  熟悉彼此 有默契，都在积极干活的话会事半功倍，
         2. 任务传达出现偏差可能性小
         3. 自己宿舍组队随时随地开组会，可能夜谈就把任务分配了  
         4. 对于不同的课程你也可以不用记住这个课谁是队友，那个课谁是队友
         5. 但是如果有人想摆烂，不想做，可能处于朋友也好等等关系上你又不好意思开口你自己吃哑巴亏，
      2. 生人：
         1. 对于陌生一点的人，相信大部分人是不好意思不干活的 
         2. 意思传达不到位理解交流有偏差 
         3. 开会受时间场地限制
      3. 生熟调配：比如你可以跟你合作负责一个板块的找熟人，其他板块的找生人，有生人有熟人，在说给陌生同学的同时也在跟自己熟人强调了么
      4. 另外就是找想干活的，有想法的，
   4. 开始项目前跟组员明确好纪律，分工没有完成遇到困难，可以尽早寻找队友帮助，如果最后因为个人原因没有完成，将会向老师申报的时候如实汇报，并将会把他的分数适当分给其他同学
   5. 所以什么是真的合作方式？我正在跟一个清华的学长在做一个项目，在跟他接触的第一次会议他就跟我描述了他自己在课程小组合作的作业上找的队友是什么样的。他和他的队友们不是上下级的关系，一个安排任务一个完成任务，所以找队友要找敢于说出自己想法的队友，而不是一个工作机器，完成分内的事情就结束掉，所以在任务开始之前可以先进行头脑风暴，每个人把自己的对于这个的想法充分讨论之后，再确定实施步骤，然后完成，这样能够让每个人都把自己想法充分说出来这才是真正的合作
