   1. DLX的字长是32位，4个字节组成，8位的寻址能力，DLX是字节可寻址的

      1. 大端存储是高字节存在低位，但是小端存储是高字节存在高位
      2. 高字节左边的在开头所以说是顺着的么
   3. <span style="color:#FF0000">DLX是大端存储（小米是小端存储）</span>

   1. 算数/逻辑运算指令
   2. 数据传送指令

   4. 浮点指令
5. 数据类型
   1. 二进制补码整数和浮点数

#### 11.6 栈--一种抽象的数据类型

1. 为了解释C语言编译器如何在存储器中分配变量，下面介绍一种重要的存储结构：栈

2. 事实上，栈的使用是贯穿于计算机科学与工程之中的,栈是一种存储结构，可以通过许多不同的方式来实现，但是栈的概念与其实现毫无关系

3. 栈的概念是说明它如何被访问其定义是最后存入栈的内容首先被取出简单地说就是后进先出（LAST IN FIRST OUR)

4. 在计算机程序设计语言中栈是抽象的数据类型的一个例子，抽象数据类型是一类存储机制这种机制是对它执行的操作所定义的而不是实现它的方式

5. 在存储器中实现栈：

   1. 当把一个元素插入栈中的时候，称之为压栈(PUSH)

   2. 当移除一个元素时，我们称之为出栈（POP)

   3. 这个栈有一组存储单元和称为栈指针的机制组成，栈指针只是了这个站的栈顶。也就是说，包含最后压入的元素的存储单元地址。每个压入的数据都被存储在4个连续的存储单元中，在这种情况下，已经存储在栈中的数据不会进行物理移动

   4. 压栈的实现

      1. R29中包含x4000 0000 即栈第一个单元的前面一个地址这说明这个栈初始为空

      2. 当把一个数据压入栈时，栈指针减4，这个数据被存储

      3. PUSH: SUBI R29,R29,#4

         ​	    SW    0(R29),R2

   5. 出栈的实现

      1. 当把一个数据出栈，栈指针加4，这个数据被存储

      2. POP: LW  R2,0(R29)

         ​	  ADDI R29,R29,#4

6. 什么时候使用栈？

   1. 就是当你的寄存器的数量不够用的时候就使用栈
   
   2. 当我们出现了函数中调用了函数的时候，R31的返回值都是不一样的，所以需要这种结构
   
   3. 在汇编语言里面写函数的时候，尽量使用栈的方式来进行书写，因为别的函数还会对这个函数进行调用，这就可能会符合函数中调用函数的情况了
   
   4. 为什么函数调用函数的时候是危险的，因为考虑到返回值需要R31的帮助，所以说R31需要进行存储，一个函数其中并没有调用其他的函数，为了防止别的函数在调用这个子例程的时候发生寄存器的覆盖，所以在写子例程的时候尽量还是要使用栈的数据结构
   
   5. 温馨提示一下：使用栈对其进行初始化的时候，使用R30往前面推，R30-4 -8 -12……
   
   6. jal指令在执行的时候会自动将下一条的指令的地址，也就是现在的PC的值加载进R31当中去

   1. 一般情况下，会有两个寄存器，

   1. 第一种的实现方式是每一个I/O设备都被分配一个存储器地址空间中的地址，
      这样就可以采用识别存储器地址的方式识别出I/O设备寄存器

   2. 第二种专门的I/O指令是指使用专门的I/O指令和I/O设备寄存器交互，
      I/O包括IN和OUT用于在通用寄存器和I/O设备之间传送数据

   3. DLX使用的是内存映射I/O地址xFFFF 0000 ~ xFFFF 00FF被分配给了I/O设备寄存器，其他的地址被分配给了存储单元

   4. 如果要从KBDR中读取一个数据，可以使用下列指令序列

   5. ```asm

      ……
      LW R1,KBDR(R0)
      LW R4,0(R1)
      ```

   6. 就默认了，xFFFF 0000 ~ xFFFF 00FF已经不再是存储单元了，他就是纯粹的给I/O设备用的了

   1. I/O设备与微处理器的步伐都是不一致的，这种特征称为异步
   2. 键盘和显示器都会有一个状态寄存器的东西，状态成为标志用来表明是否输入了一个字符（键盘）对于显示器来说就是最近传送给显示器的字符是否已经被显示了

   1. 这种通过处理器周期性地检查状态来判断是否执行I/O叫做轮询

## 第十三章 自陷历程和中断

#### 13.1 DLX自陷历程

1. 系统调用

   1. 数据寄存器
      1. 显示器的数据寄存器用于显示提示符（R4已经有了数据）
      2. 键盘数据寄存器是用于让系统知道到哪里去寻找数据（要往R4里面输入数据）
   2. 状态寄存器
      1. 显示器状态寄存器是为了让程序知道什么时候可以显示提示符中的下一个字符
      2. 键盘状态寄存器让程序知道什么时候有人输入了一个字符
   3. 键盘输入和执行程序之间的异步关系
      1. 硬件寄存器是有特权的
      2. 更简单更安全的解决方案就是使用TRAP指令
      3. 在某些任务中，用户程序通过调用TRAP指令来时操作系统完成这个工作
      4. 处理TRAP指令的请求，然后把控制权返回给用户程序

2. TRAP机制

   1. 服务例程
      1. 服务例程是操作系统里面的一部分，是代表用户程序去执行的一组程序
      2. DLX一共是有256个服务例程
      3. x06读取字符
      4. x07输出字符printf("%c",word)
      5. x08将R4所指的地址开头的一个字符串输出到显示器上每个字符占用一个存储单元直到00为止puts(string)
      6. x09  scanf("%c",&word)
      7. x0A gets(string)
      8. x00 HALT 终止程序，停止执行
   2. TRAP向量表
      1. TRAP向量表是一张包括了256个服务例程的<span style="color:#AABBFF">起始地址</span>的表，每个起始地址需要占用4个连续的存储单元
      2. 所以一共需要256$\times$4 = 1024个字节
      3. 所以这些起始地址存储在了x0000 0000 - x0000 03FF(256$\times$3+15$\times$16+15 = 1024)
      4. 每个服务例程包括数据区和代码区
      5. TRAP指令让用户程序把控制圈交给了操作系统，然后又把控制圈给要了回来
      6. 服务例程必须有一种可以把控制权交还给程序的机制

3. TRAP指令

   1. TRAP指令需要提供两方面的工作执行服务例程
      1. 根据TRAP向量，改变PC的值变成相应的服务例程的首地址
      2. 提供一个返回调用TRAP指令的程序的路径，即链接
   2. TRAP指令由两个部分组成:TRAP的操作码（110000）和TRAP向量[25:0]
   3. 四个步骤
      1. 将TRAP指令进行如下处理TRAP向量==>32位\==>左移2位形成一个地址这个地址加载到MAR
      2. 读取TRAP向量表中的MAR地址相应的数据，这个数据也是一个地址加载到MDR
      3. R31被加载为PC当前的内容这将给程序提供一个返回路径(R31中会记录当前的要执行的下一条命令的地址)
      4. 然后把MDR加载到PC中去，进行TRAP指令的读取和执行
   4. 总结
      1. 先把TRAP向量进行解读，得到TRAP向量表中的相应的地址
      2. 然后把这个地址加载到MDR中去
      3. R31保存这个PC
      4. PC改成MDR中的数值相应位置代码的执行

4. 完整的机制

   1. 在计算机执行指令的过程中改变控制流的方式就是在当前指令的执行阶段改变PC中的内容

   2. 这样在下一个取指令阶段读取的将是改变了方向的地址

   3. TRAP服务以JR R31结束如果R31在执行服务例程期间没有改变，他将会包含原始用户程序中紧跟着TRAP指令之后的那条指令的地址

#### 13.2 中断驱动的I/O

1. 中断驱动的I/O概述

   1. 生成中断信号的产生
   2. 处理由该中断信号发出的请求

2. 中断信号的产生

   1. I/O设备需要服务
   2. 设备必须有权去请求服务
   3. 来自设备的中断请求
      1. 在讨论轮询的时候已经描述了第一个元素，他就是KBSR或者DSR的就绪位
         也就是说如果I/O设备是键盘，如果有人输入了一个字符，他就需要服务，
         如果I/O设备是显示器，相关电路已经成功地完成了最后一个字符的显示，它就需要服务在这两种情况下当相应的就绪位变成1的时候，他们就需要服务
      2. 第二个元素是一个中断允许位，它可以被设置为0或者1，这取决于是否给I/O设备权力去请求服务。在大多数的I/O设备里IE是设备转台寄存器的一部分
      3. 如果中断允许位被设置为0，那么不管就绪位是否被设置为1，I/O设备都不能中断处理器，在这种情况下，程序需要轮询I/O设备确认它是否就绪
         1. 如果中断允许位被设置为1，那么中断驱动I/O就被允许了在这种情况下一旦用户按下某个键[0]就被设置成1这样与门输出为1结果I/O设备就生成了一个IRQ信号
      4. 只要有中断设备产生了IRQ信号那么就一定会产生INT信号
      5. 一旦产生了IRQ信号，CAUSE寄存器的相应位就会被设置为1，
         原因寄存器是一个特殊寄存器，只有在特权模式下才能够访问
      6. 为测试中断信号而增加的逻辑是将存储结果返回到取指令阶段的最后一步取代为存储结果并检测I/O信号如果I/O信号为0那么与往常一样返回取指令阶段，开始下一条指令的处理。如果INT信号是1，那么控制单元把PC加载成x8000 1000执行操作系统的中断服务例程处理由该信号发出的中断请求
      7. 程序状态是程序影响的所有资源所包含的内容的瞬态图，它包括作为程序一部分的存储的单元的内容，它还包括两个非常重要的寄存器:PC&SR
      8. PC包含了下一条要执行的指令的地址必须保存起来DLX的特殊寄存器EPC用于保存中断发生时的PC

3. 中断服务例程

   1. 按照中断需要响应的紧急程度的不同设定不同中断具有不同的中断优先级

   2. 在处理高优先级中断时往往会屏蔽部分或全部低优先级中断

      中断嵌套 :   	程序==>A\==>B\==>A\==>程序

   3. 串行处理和嵌套处理

      1. 若采用串行处理在运行一个中断的时候是禁止再响应新的中断们这可以通过屏蔽中断来实现，新的中断只能等待到正在运行的中断运行完毕才能获得响应
      2. 中断处理的时候还能响应中断，因为每嵌套一层中断会产生较大的系统开销因此还要规定允许嵌套的层数不要太多一般不超过三个为宜
         1. 中断优先级
            1. 为了让I/O设备成功地停止处理器，开始中断请求处理请求的优先级必须比他希望中断的程序要高
            2. 0-5数字越高等级越高
            3. 速度越高的设备优先级越高
            4. SR[15:10]代表6个硬件中断屏蔽位SR[9:8]代表两个软件中断的中断屏蔽位于CAUSE[15:8]一一对应，从左到右优先级依次降低
            5. 原因寄存器中的中断未决位CAUSE要等到相应的屏蔽位为1的时候才能引起中断处理器的处理
         2. 中断服务例程需要完成的任务如下
            1. 对原因寄存器中的中断未决位CAUSE和状态寄存器的中断掩码位做逻辑与运算，看发生了哪些中断
            2. 如果有多个终端的产生那么先处理左边的中断，因为左边的中断的优先级更高

4. 总结一下

   1. 服务产生
      1. 如果现在的IO是一个键盘那么只要是输入了一个字符，那么它就需要服务
      2. 如果现在的IO是一个显示器那么只要是输出完了一个字符，那么它就需要服务
   2. 中断允许位
      1. 第一关SR
         1. 所有设备的XXSR[1]可以被SR[0]同时改写
         2. SR[0]=0那么就不允许任何的设备中断处理器
         3. SR[0]=1那么这关算是过去了 
      2. 第二关XXSR[1]
         1. 如果XXSR[1] = 0那也产生不了这个IRQ信号
         2. 当XXSR[1] = 1的时候才能产生IRQ信号，这关算是过去了
      3. 注意：只要有设备产生了IRQ,就会产生INT
      4. 第三关CAUSE[i]和SR[i]
         1. 当你产生了一个IRQ信号的时候，CAUSE[i]就会变成1
         2. 接下来就是将SR[i]&CAUSE[i]进行判断
            1. 第一有没有1
            2. 第二是否存在多个1
            3. 第三从左到右就是从优先级从高到低进行处理
      5. 到了这里就已经可以处理了
      6. 关于保存状态的问题
         1. SR[0] ==>判断能否被打断
         2. SR[1] ==>判断是否是特权模式（特权模式是0，反之则是1）
         3. 如果这时候程序再一次被打断了，
            1. SR[0]的值会放在SR[2]中进行保存
            2. SR[1]的值会放在SR[3]中进行保存
         4. MOVSI是把特殊寄存器的值拉出来给证书寄存器的特殊指令
         5. RFE指令是令PC返回EPC值的指令
         6. SR ==>0C
         7. CAUSE ==> 0D
         8. EPC ==> 0E
   3. 中断结束返回的指令是REF指令
