## 一、校园作业提交通知系统设计题（25 分）

某课程平台需要实现“作业提交通知系统”，需求如下：

1. 学生提交作业后，系统需要**自动通知**教师、助教、课程管理员
2. 不同课程的通知方式不同：
   - 有的课程发邮件
   - 有的课程发短信
   - 有的课程发站内信
3. 通知前需要先统一完成日志记录、权限校验、消息内容组装
4. 后续可能新增企业微信通知、钉钉通知

1. 本题至少组合使用 **观察者模式** 和 **策略模式**

   - 如何划分“提交作业”和“通知订阅者”之间的职责？
   - 如何保证以后新增通知方式时尽量少改原代码？
   - 这个设计体现了哪些面向对象设计原则？
4. 给出核心类设计，至少包含：
   - 作业提交者/被观察者

## 二、数据库备份系统设计题（25 分）

学校机房要做一个“数据库定时备份系统”，需求如下：

1. 系统支持 `MySQL`、`Oracle`、`SQL Server`
2. 每种数据库的连接方式不同，但备份整体流程基本一致：
   - 打开连接
   - 身份认证
   - 导出数据
   - 压缩
   - 关闭连接
3. 备份任务可以被调度中心放入队列，支持“立即执行”“撤销任务”“延迟执行”
4. 后续可能新增 PostgreSQL

### 要求

1. 本题至少组合使用 **模板方法模式** 和 **命令模式**
2. 分析：
   - 哪部分适合做成固定流程？
   - 哪部分适合封装成独立命令对象？
3. 回答下面问题：
   - 如果以后新增数据库类型，哪些类应该扩展，哪些类不应该修改？
   - 这个设计体现了哪些设计原则？
4. 给出核心类设计，至少包含：
   - 抽象备份模板类
   - 至少两种具体数据库备份类
   - 命令接口
   - 至少两种命令对象
   - 调用者/调度者
5. 写出“调度中心下发备份命令并执行”的关键流程

## 三、跨平台绘图系统设计题（25 分）

某绘图软件需要支持不同画笔、不同颜色、不同图形对象，需求如下：

1. 画笔有：小号、中号、大号
2. 颜色有：红色、蓝色、黑色
3. 图形有：圆形、矩形、文本框
4. 图形可以单独绘制，也可以组合成复杂面板统一绘制
5. 某些图形需要动态增加功能，例如：
   - 边框高亮
   - 阴影
   - 动画闪烁

### 要求

1. 本题至少组合使用 **桥接模式** 和 **组合模式**
2. 如果你认为还应引入 **装饰者模式**，请进一步说明其职责
3. 回答下面问题：
   - 画笔维度和颜色维度为什么适合分离？
   - 为什么“复杂面板”不适合仅靠继承实现？
   - 动态附加“阴影/高亮”等能力时，为什么装饰者优于直接修改原类？
4. 给出核心类设计，至少包含：
   - 抽象画笔
   - 颜色接口/抽象类
   - 图形抽象类
   - 叶子图形
   - 容器图形
   - 至少一个装饰者类
5. 写出“一个带阴影效果的大号红笔绘制组合图形”的关键流程

## 四、校园支付聚合网关设计题（25 分）

学校要做一个统一支付网关，接入多个第三方支付平台，需求如下：

1. 前端只调用统一的支付接口
2. 后端要兼容：
   - 微信支付
   - 支付宝
   - 校园卡支付
3. 第三方平台接口风格不同：
   - 有的平台是 `pay()`
   - 有的平台是 `execute()`
   - 有的平台要求先创建请求对象再提交
4. 系统还要支持根据支付场景创建不同的一组对象，例如：
   - PC 支付场景
   - 移动端支付场景
   - 自助终端支付场景
5. 后续可能扩展退款、查询、对账

### 要求

1. 本题至少组合使用 **适配器模式** 和 **抽象工厂模式**
2. 回答下面问题：
   - 统一支付接口和第三方支付接口之间为什么会产生适配需求？
   - 为什么“不同支付场景的一组对象”适合用抽象工厂？
   - 如果未来接入新的支付平台，如何尽量满足开闭原则？
3. 给出核心类设计，至少包含：
   - 统一支付接口
   - 至少两种第三方支付类
   - 至少两种适配器
   - 抽象工厂
   - 至少两种具体工厂
4. 分析本题体现的设计原则，至少写出 3 条
5. 写出“移动端场景下调用统一接口完成微信支付”的关键流程

## 评分参考提示

你做题时可以注意以下得分点：

1. 模式是否真的“用对了”，而不是只写名字
2. 模式之间的职责边界是否清晰
3. 是否真正体现了开闭原则、单一职责原则、依赖倒置原则等
4. 类之间的依赖关系是否合理
5. 关键流程是否能闭环运行
