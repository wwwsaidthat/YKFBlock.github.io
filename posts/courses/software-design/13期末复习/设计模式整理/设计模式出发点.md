# 创建型模式

1. 简单工厂模式：Simple Factory Pattern / Static Factory Method

   1. 模式要点：你只需要传入一个正确的参数你就可以获得对应的对象，而无需知道创建的类的具体细节
   2. 模式优点：降低系统的耦合度 
   3. 模式缺点：工厂压力大，增加新的类需要修改判断逻辑，违背开闭原则
   4. 使用场景：工厂负责创建的对象比较少，而且客户传入参数对于如何创建并不关心
   5. 实现注意：工厂的创建方法必须是**静态方法** 单个工厂 单个产品
   6. 简化：把抽象产品类作为工厂

2. 工厂方法模式：Factory Method Pattern 工厂方法模式 / Virtual Constructor 虚拟构造器具 / Polymorphic Factory 多态工厂模式

   1. 模式要点：核心的工厂类不再负责所有产品的创建，而是将具体的创建工作交给子类 单个抽象产品类

   2. 模式优点：工厂方法也可以让用户在不知道具体产品类的类名的前提下创建对象，工厂方法模式可以允许系统在不修改工厂角色的情况下引进新产品

   3. 模式缺点：增加新的类需要编写具体产品类和对应的工厂类，类的个数成对增加，增加了系统的复杂程度，增加了系统的抽象性和理解难度

   4. 使用场景：一个类不知道它所需要的对象的类，一个类通过其子类来指定创建哪个对象，将创建对象的任务委托给多个工厂子类中的某一个， 客户端在使用时可以无须关心是哪一个工厂子类创建 产品子类，需要时再动态指定。

   5. 实现注意：定义工厂和产品必须使用**抽象层** 单个抽象工厂 多个工厂 单个抽象产品 多个产品 分别对应

   6. java反射机制是指 **在程序运行的时候获取已知名称的类或者已有对象的相关信息的一种机制**

      1. 可通过Class类的forName()方法返回带有给定字符串名的类或者接口相关联的Class对象

      2. 通过newInstance()方法创建次对象所表示的类的一个新的实例，**通过一个类名字符串得到类的实例**

      3. ```java
         Class c = Class.forName("String");
         Object obj = c.newInstance();
         return obj;
         ```

3. 抽象工厂模式：Abstract Factory Pattern / Kit模式

   1. 模式要点：创建一系列相关或者相互依赖对象的接口，而无需指定它们具体的类                           

      多个抽象产品类 多个抽象产品 单个抽象工厂类 一个工厂对应多个抽象产品 

   2. 模式优点：一个工厂可以提供多个产品对象，而不是单一的产品对象 高内聚低耦合‘

   3. 模式缺点：增加新的产品族或者新的工厂比较容易的，但是新增新的产品结构会比较复杂（每个工厂都需要增加一个新的声场方法）不太符合开闭原则的地方，但是新增（工厂/产品族）是完全符合开闭原则的

   4. 使用场景：

      1. 一个系统不应当依赖于产品类实例如何被创建、组合和表达的细节，而且有多于1个产品族
      2. 属于同一个产品族的产品在一起使用，所有的产品以同样的接口出现，从而使得客户端不再依赖具体实现
      3. 多个位于不同产品等级结构中属于不同类型的具体产品的时候需要使用抽象产品工厂

   5. 产品等级结构 & 产品族

      1. 产品等级结构就是产品的继承结构：比如一个产品是电脑，有huawei的 apple的 alient的 联想的 同类产品不同类型 产品登记结构
      2. 产品族是指：apple家一个产品是电脑 一个产品是手表apple watch 一个产品是apple mini主机，一个产品是iphone 一个产品是ipad就是指一个工厂的多种产品等级结构下面的产品

   6. 区别：工厂模式是针对一个产品等级结构，抽象工厂模式是针对多个产品等级结构

总结三种工厂模式之间的退化关系

	抽象工厂当只有一个产品族的时候就会退化成工厂模式
	工厂模式将具体工厂和抽象工厂合并成一个工厂负责所有产品的生成的时候就会退化成简单工厂模式
