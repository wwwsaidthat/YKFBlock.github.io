# 设计模式代码实现

# 创建型模式

1. 简单工厂模式：工厂的创建方法必须是static的，创建产品的时候不允许出现这个new 直接用factory的创建方法，直接用类的创建方法就行不用单独创建一个对象

   ```java
   class SimpleFactory{
     public SimpleFactory(){}
     
     public static Product createProduct(String type){
       Product product;
       swich(type)
       case "jiucai":
       	product = new ConcreteProduct1();
       	break;
       case "meat":
       	product = new ConcreteProduct2();
       	break;
       default:
       	product = null;
       	break;
       return product;
     }
   }
   
   abstract class Product{}
   class ConcreteProduct1 extends Product{}
   class ConcreteProduct2 extends Product{}
   
   public class Main{
     public static void main(String[] args){
       SimpleFactory factory = new SimpleFactory();
       Product product = factory.createProduct("jiucai");
      	
     }
   }
   ```
   
   **总结：简单工厂必须是static方法，而且注意一下调用的时候是类直接调用，工厂和抽象工厂的创建方法就不用是static的了**

4. 单例模式：保证一个类只有一个实例，并提供一个访问他的全局访问点

   ```java
   public class Main{
     public static void main(String[] args){
       SinglePattern s1 = SinglePattern.getInstance();
       SinglePattern s2 = SinglePattern.getInstance();
       SinglePattern s3 = SinglePattern.getInstance();
     }
   }
   class SinglePattern{
     private static SinglePattern instance;

       return instance;
     }
   }
   ```

5. 原型模式：用原型实例制定创建对象的种类，并且通过复制这些原型创建新的对象

   ```java
   public class Main{
     public static void main(String[] args){
       PrototypeA pA = new PrototypeA(1,1.1);
       PrototypeA pA1 = pA.clone();
       System.out.println(pA1.id);
       System.out.println(pA1.price);
     }
   }
   
   interface Prototype{
     public abstract Prototype clone();
   }
   class PrototypeA implements Prototype{
     public int id;
     public double price;
     public PrototypeA(int id,double price){
       this.id = id;
       this.price = price;
     };
     
     @Override 
     public PrototypeA clone(){
       return new PrototypeA(id,price);
     }
   }
   
   ```

   

6. 建造者模式：将一个复杂对象的构建与他的表示分离，使得同样的构建过程可以创建不同的表示

```java
public class Main{
  public static void main(String[] args){
    Builder builder = new ConcreteBuilder();
    builder.buildPart();
    Product product = builder.getResult();
    product.show();
  }
}

class Product{
	List<String> parts;
  public void Add(String part){
    parts.add(part);
  }
  public void show(){
    System.out.print("Composed by:");
    for(String part:parts){
      System.out.print(part + " ");
    }
    System.out.println();
  }
}

abstract Builder{
  public abstract void buildPart();
  public abstract Product getResult();
}
class ConcreteBuilder extends Builder{
  Product product= new Product();
  @Override
  public void buildPart(){
    product.Add("A");
    product.Add("B");
    product.Add("C");
  }
  @Override
  public Product getResult(){
    return product;
  }
}

```





# 结构型模式

1. 适配器模式：将一个类的接口转换成客户希望的另外一个接口 为了从不兼容到兼容

   Adapter 继承自被适应的类 拥有去适应的类作为对象

   

   ```java
   public class Main{
     public static void main(String[] args){
       
       USB usb = new Adapter(new TYPEC);
       usb.Request();
     }
   }
   
   class TYPEC{
     public void SpecificRequest(){
       System.out.println("type_c charging");
     }
   }
   class USB{
     public void Request(){
       System.out.println("usb charging");
     }
   }
   class Adapter extends USB{
     private TYPEC typec = new TYPEC();
     public Adapter(TYPEC t){
       this.typec = t;
     }
     @Override
     public void Request(){
       typec.SpecificRequest()
     }
   }
   ```

2. 桥接模式:将抽象部分和实现部分分离，是他们都可以独立地变化 主要是为了分离维度 多个维度进行扩展

   ```java
   public class Main{
     public static void main(String[] args){
       Product product = new ProductA();
       product.setName("zhangsan");
       product.setColor (new Red());
       product.Operation();
     }
   }
   
   abstract class Product{
     private String name;
     protected Color color;
     
     public void setName(String name){
       this.name = name;
     }
     public String getName(){
       return this.name;
     }
     public void setColor(Color color){
       this.color = color;
     }
     public Color getColor(){
       return this.color;
     }
     public abstract void Operation();
     
   }
   class ProductA extends Product{
     @Override
     public void Operation(){
       color.OperationImp(this.getName)
     }
   }
   interface Color{
     public void OperationImp(String name);
   }
   class Red implements Color{
     @Override
     public void OperationImp(String name){
       System.out.println(name+"red");
     }
   }
   ```

   两条腿走路 继承一个 实现一个 实现的是成员变量的接口

3. 组合模式:将对象组合成树型结构用来表示部分-整体的层次结构

   ```java
   import java.util.List;
   import java.util.ArrayList;
   
   public class Main{
    public class void main(String[] args){
      AbstractFile root  = new Folder("root");
      AbstractFile folderA = new Folder("A");
      AbstractFile fileB = new File("B");
      root.add(folderA);
      root.add(fileB);
      
      root.printName();
    } 
     
    static void print(AbstractFile root){
      root.printName();
      
      List<AbstractFile> files = root.getChildren();
      for(AbstractFile file: files){
        print(file);
      }
    } 
   }
   
   abstract class AbstractFile{
     protected String name;
     
     public void printName(){
       System.out.println(name);
     }
     
     public abstract boolean add(AbstractFile file);
     public abstract boolean remove(AbstractFile file);
     public abstract List<AbstractFile> getChildren();
   }
   
   class Folder extends AbstractFile{
     private List<AbstractFile> files = new ArrayList<>();
     public Folder(String name){
       this.name = name;
     }
     @Override
     public boolean add(AbstractFile file){
       
       return files.add(file);
     }
     @Override
     public boolean remove(AbstractFile file){
       
       return files.remove(file);
     }
     @Override
     public List<AbstractFile> getChildren(){
       return files;
     }
   }
   
   class File extends AbstractFile{
     public File(String name){
       this.name = name;
     }
     @Override
     public boolean add(AbstractFile file){
       return false;
     }
     @Override
     public boolean remove(AbstractFile file){
       return false;
     }
     @Override
     public List<AbstractFile> getChildren(){
       return null;
     }
   }
   ```

   component类由leaf和composite两个子类继承 leaf可能会是composite的成员变量

继承一个 拥有一个

1. 适配器模式 继承被适配方，拥有去适配别人的一方
2. 装饰器模式 继承父类，拥有父类
3. 桥接模式 继承抽象类，拥有接口（抽象父类就有）
4. 组合模式 继承父类，复杂子类拥有简单子类
5. 享元模式 非共享子类继承父类 拥有共享子类
6. 代理模式 继承父类，拥有兄弟类
7. 外观模式最简单，但是没有继承一个拥有一个，就是封装一系列类，只有一系列的拥有

2. 中介者模式:用一个中介对象封装一系列的对象交互

   ```java
   import java.util.List;
   import java.util.ArrayList;
   public class Main {
       public static void main(String[] args) {
           Mediator mediator = new ConcreteMediator();
         	Person p1 = new Person1("Zhang San",mediator);
         	Person p2 = new Person1("Li Si",mediator);
         	Person p3 = new Person1("Wang Wu",mediator);
         	Person p4 = new Person1("Yao Kaifang",mediator);
         	Person p5 = new Person1("Du Xinyao",mediator);
           p1.sendMessage("Li Si","Time to get up.");
         	p4.sendMessage("Du Xinyao","I love you.");
       }
   }
   abstract class Mediator{
       public abstract void SendMessage(String to,String message);
       public abstract void Add(Person person);
   }
   
   class ConcreteMediator extends Mediator {
     List<Person> persons = new ArrayList<>();
     @Override
     public void Add(Person person){
       persons.add(person);
     }
     @Override
     public void SendMessage(String to,String message){
       for(int i = 0 ; i < persons.size() ;i++){
         Person p = persons.get(i);
         if(to.equals(p.name)){
           p.getMessage(message);
           break;
         }
       }
     }
     
   }
   abstract class Person{
     protected String name;
     protected Mediator mediator;
     public abstract void sendMessage(String to,String message);
     public abstract void getMessage(String message);
   }
   
   class Person1 extends Person{
     public Person1(String name,Mediator mediator){
       this.name = name;
       this.mediator = mediator;
       mediator.Add(this);
     }
     @Override
     public void sendMessage(String to,String message){
       mediator.SendMessage(to,message);
     }
     @Override
     public void getMessage(String message){
        System.out.println(name + " get message:" + message);
     }
   }
   ```
